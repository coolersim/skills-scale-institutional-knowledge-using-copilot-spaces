# OctoAcme Project Management Docs

This directory is the central landing page for OctoAcme's project management process documentation. It provides a quick reference for onboarding and day-to-day navigation across all process docs.

---

## Purpose

These docs define how OctoAcme plans, executes, and closes projects. They are intended to help new and existing team members understand our methodology, roles, key artifacts, and communication expectations.

---

## Core Principles

- **Customer-first:** Prioritize customer value and usability.
- **Iterative delivery:** Ship frequent, testable increments.
- **Clear ownership:** Each project has a named Project Manager (PM) and Product Lead.
- **Data-informed decisions:** Measure impact and iterate based on evidence.
- **Psychological safety:** Encourage feedback and continuous learning.
- **Risk management:** Ongoing risk tracking and timely escalation.
- **Continuous improvement:** Retrospectives and action items after every project.

---

## Roles

| Role | Responsibility |
|------|---------------|
| **Project Manager (PM)** | Coordinates delivery, schedules, risks, and communications. |
| **Product Manager (PdM)** | Defines outcomes, prioritizes the backlog, and measures success. |
| **Developers** | Implement features, collaborate on design, testing, and code reviews. |
| **QA / Testing** | Validate quality and acceptance criteria. |
| **Stakeholders** | Provide inputs, approvals, and strategic direction. |

See [Roles & Personas](./octoacme-roles-and-personas.md) for detailed persona definitions.

## Key Artifacts

- Project Charter / One-pager
- Roadmap and Release Plan
- Sprint / Iteration Backlog
- Acceptance Criteria & Definition of Done
- Risk Register
- Retrospective notes and action items

## Project Lifecycle

### 1. Initiation
- Define the problem statement, goals, and key stakeholders.
- Establish a high-level timeline and initial scope.
- Produce the Project Charter.

→ [Project Initiation Guide](./octoacme-project-initiation.md)

### 2. Planning
- Break scope into milestones, tasks, and sprint deliverables.
- Identify dependencies, resources, and risks.
- Finalize the Roadmap and Release Plan.

→ [Project Planning](./octoacme-project-planning.md)

### 3. Execution & Tracking
- Build, test, review, and iterate in short cycles.
- Track progress against milestones; surface blockers early.
- Maintain the Sprint Backlog and status updates.

→ [Execution & Tracking](./octoacme-execution-and-tracking.md)

### 4. Risk Management & Communication
- Maintain an active Risk Register; escalate as needed.
- Communicate project status to stakeholders on a regular cadence.

→ [Risk Management & Communication](./octoacme-risks-and-communication.md)

### 5. Release & Deployment
- Coordinate release readiness, deployment steps, and go/no-go decisions.
- Verify the release and communicate to stakeholders.

→ [Release & Deployment Guide](./octoacme-release-and-deployment.md)

### 6. Close & Retrospective
- Capture learnings, celebrate wins, and document action items.
- Archive project artifacts for future reference.

→ [Retrospective & Continuous Improvement](./octoacme-retrospective-and-continuous-improvement.md)

## Communication Cadence

| Cadence | Participants | Purpose |
|---------|-------------|---------|
| Weekly sync | PM + PdM | Alignment on priorities and blockers |
| Twice-weekly standups | Delivery team | Progress updates and blocker resolution |
| Monthly updates | Stakeholders | Status, risks, and upcoming milestones |
| Ad-hoc escalations | As needed | Urgent issues requiring immediate attention |
